AI19｜模型平台時代結束，Google 的 Agent 元年正式開幕

作者 Jin · 2026-05-16

Google Cloud Next 2026 做了一件震驚業界的事：把經營五年的旗艦 ML 平台 Vertex AI 直接整併改名，宣告「模型平台時代」的終結。這篇文章從這場發表會拆解 Google 的戰略邏輯，比較 AWS、微軟、Google 三大雲端廠商的 Agent 平台差異，並探討這個轉變對台灣企業與工程師的實際意義。

一場沒人預料到的告別式

如果你有在追雲端 AI 的動態，Google Cloud Next 2026 大概是近幾年最值得細看的一場發表會。不是因為它發布了多震撼的新模型，而是因為它做了一件很多人沒有預料到的事——Google 親手把跑了五年、堪稱自家 ML 旗艦的 Vertex AI，整個改名整併進一個叫做 Gemini Enterprise Agent Platform 的新平台。

這集 Podcast 我用一個比較直白的角度來聊這件事：這場發表會，其實是 Vertex AI 的告別式，也是 Google 正式宣告 Agent 元年的起點。

為什麼要在這個時間點動刀？

Vertex AI 不是一個失敗的產品，它是 Google 在企業 MLOps 市場最重要的入場券。那 Google 為什麼要在這個時間點，親手埋葬它？

答案其實不難理解：市場的競爭維度已經整個位移了。過去幾年，各家雲端廠商比的是「我的模型有多強」；但現在，光有強模型已經不夠，企業真正在意的是——我能不能快速把 AI 接進現有流程、讓它自動做事、還能管控它的行為。這就是 Agent 平台的核心價值。

Gemini Enterprise Agent Platform 的架構圍繞四個面向展開：建構、擴展、治理、最佳化。這四個詞看起來很制式，但對應的其實是企業 IT 部門每天在煩惱的真實痛點——怎麼建 Agent、怎麼跟現有系統串接、出了事誰負責、跑起來的成本能不能壓下來。Google 這次的佈局，就是要在這四個環節都給出答案。

三朵雲的 Agent 邏輯：樂高、橋樑、一條龍

這集我用了一個比喻來區分 AWS、微軟、Google 三家的 Agent 平台哲學，我覺得這是整集最值得記起來的框架：

  • AWS 是樂高：元件極其豐富，你可以自由組合，但你得自己想清楚要拼什麼、怎麼拼。適合有強大工程團隊、想要最大彈性的組織。
  • 微軟是橋樑：它的優勢在於跟 Office 365、Teams、Azure AD 這些企業既有系統的深度整合。如果你的公司已經是微軟生態系，Agent 接進去的摩擦力是最低的。
  • Google 是一條龍：從模型、到平台、到基礎設施，Google 希望你在它的體系裡一路走到底。Gemini 模型、Agent 平台、TPU 算力，都是自家的。

選平台不是選最強的，是選跟你現有架構最契合的。這個判斷邏輯，對正在評估雲端 AI 策略的台灣企業來說，應該是比任何跑分數字都更實用的參考。

TPU 拆分這件小事，藏著一個大訊號

發表會還有一個細節值得特別關注：Google 這次首次把 TPU 拆分成訓練卡與推論卡兩個獨立產品線。這個動作看起來是硬體規格的調整，但背後傳遞的訊號是——推論工作負載的規模已經大到需要專門的硬體來對應，而這正是 Agent 大量部署之後必然會面臨的基礎設施問題。Google 在硬體層面提前卡位，是整個 Agent 平台戰略的一部分，不是獨立事件。

對工程師與企業，這個視窗不會永遠開著

AI 的競爭重心，正在從「模型層」整個搬到「平台層」。這對台灣的企業和工程師來說，其實是一個相對有利的時間窗口——因為平台層的競爭還在成形，各家的生態系都還沒有完全鎖死，現在進場評估、累積實作經驗，比等到市場格局確定之後再跟進，成本要低得多。

但這個視窗不會永遠開著。等到 Agent 平台的標準慢慢收斂、各家的護城河越挖越深，選擇空間只會越來越小。

結語

Vertex AI 的改名，不只是一個品牌決策，它代表的是整個企業 AI 市場的典範轉移。這集 Podcast 試著把這個轉移的脈絡講清楚，也希望幫你在評估自己的 AI 策略時，多一個有用的參照框架。如果你正在思考公司的雲端 AI 路線，或者只是想搞懂這場發表會到底在說什麼，這集值得完整聽一遍。
