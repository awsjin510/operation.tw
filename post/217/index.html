EP66｜狂掃 15 張 AWS 證照、拿到黃金夾克，雲端架構師 Tony 的轉職實戰心法

2025-04-09

從企業管理到雲端架構師，Tony 用五年時間考下 15 張 AWS 認證、成為 AWS 大使，還拿到象徵全球頂尖認可的 Golden Jacket。這集 Podcast 他不藏私分享：怎麼從零開始學雲端、SA 的角色究竟在做什麼、AI 時代新人該如何突圍，以及那句讓他一路走來的座右銘。

如果你曾經懷疑過「我不是本科系，能不能跨進雲端產業？」——Tony 的故事，或許正是你需要聽的那一集。

在這集《操作一下》Podcast 中，我們邀請到資深雲端架構主管 Tony，他擁有超過五年的 AWS 實戰經驗、15 張 AWS 認證，同時是 AWS 大使與 AWS Community Builder 社群促進者，更是少數獲得傳說中 Golden Jacket 的台灣人之一。他的起點，不是資工系，而是企業管理。

從微軟實習生到雲端架構師，轉職的契機在哪裡？

Tony 最初在微軟擔任實習生，負責推廣 Windows 10，那段經歷讓他提早嗅到公有雲正在崛起的趨勢。後來透過研究所同學的引薦，他進入雲端相關產業，一開始擔任的是產品行銷的角色——不是工程師，而是負責寫技術文章、溝通產品。

就是這個「寫文章」的契機，讓他開始大量閱讀 AWS 技術文件，技術的濃厚興趣也就此被點燃。有趣的是，Tony 在還沒轉職為技術職之前，就已經在產品行銷期間考下了整整十張 AWS 認證。這件事本身就說明了一件事：學習雲端，不需要等到你已經是工程師。

AWS 解決方案架構師（SA）到底在做什麼？

很多人對「架構師」的想像是整天畫系統圖、寫程式，但 Tony 在節目中點出了一個關鍵：SA 的核心價值，是扮演客戶與公司業務之間的技術溝通橋樑。

這個角色需要的能力組合包括：

  • 足夠扎實的技術基礎，能看懂客戶的架構問題
  • 良好的客戶溝通與需求搜集能力
  • 一定程度的專案管理能力

換句話說，SA 不只是技術職，更是一個需要軟實力的角色。Tony 特別強調，溝通、團隊合作、以及把複雜技術「翻譯」成客戶能理解的語言，往往是決定一位 SA 能走多遠的關鍵。

考 AWS 認證，從哪裡開始？

針對想入門 AWS 認證的朋友，Tony 給出了很實際的建議：先從 AWS Skill Builder 平台上的免費線上課程開始，不需要花大錢買課程，官方資源就已經相當豐富。

想往進階認證（如 SAA，Solutions Architect Associate）邁進的話，光讀理論還不夠，Tony 建議要多做 workshop 和 lab 練習，因為進階題目非常注重你對實際使用情境的理解，而不只是背服務名稱。

至於傳說中的 Golden Jacket，條件只有一個：取得所有 AWS 認證。Tony 透過參與 AWS Community Builder 計畫中的 Golden Jacket program 完成了這個目標，這件外套代表的是全球級別的技術認可，也是他多年持續學習的具體見證。

AI 時代，雲端新人還有機會嗎？

這個問題幾乎是每個想入行者心中最大的焦慮。Tony 坦言，AI 的快速發展確實可能讓企業更傾向「用資深人員搭配 AI 工具」，對中途轉職或剛畢業的新人來說，競爭門檻確實在提高。

但他也給出了應對思路：反過來把 AI 當成學習工具，並且在面試中展現出你不只是「會用工具」，而是真的理解技術背後的知識點與實作邏輯。表面的操作誰都能做，但對原理的理解，才是讓你和 AI 工具區隔開來的核心競爭力。

給想入行者的路徑建議

Tony 建議，如果你是初階者，不要一開始就鎖定「架構師」的職稱，而是先從雲端工程師做起，踏踏實實熟悉雲端觀念與服務建置，累積兩到三年的實戰經驗後，再往架構師方向發展，這樣的路徑會更穩固。

對於還在學的同學，他也鼓勵在學期間就開始修習相關課程，甚至把考取證照列為學期目標之一。非本科系的朋友更不用害怕，雲端產業對軟實力的重視程度，和技術能力是並駕齊驅的。

結語：開始，才能厲害

節目最後，Tony 分享了他最喜歡的一句話：「你不需要很厲害才開始，你要開始才能厲害。」

這句話，或許也是整集最值得帶走的東西。不管你現在在哪個位置，雲端這條路的起點，永遠都在你決定打開第一份文件、報名第一堂課的那一刻。如果你對雲端轉職或 AWS 認證有興趣，這集 Podcast 絕對值得完整聽一遍，Tony 的親身經歷會告訴你，這條路走得通。
